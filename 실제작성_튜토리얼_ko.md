{{ prolog }}

스크립트와 프론트엔드 화면을 구분했으니, 이제 실제로 작성할 수 있습니다.
술집 도우미가 어떤 기능을 제공하는지 모르겠다면 [술집 도우미 문서](https://n0vi028.github.io/JS-Slash-Runner-Doc/)를 먼저 확인하세요.

## 시작 전

아래 내용은 먼저 `예시 캐릭터 카드`, `示例/脚本示例`, `示例/前端界面示例`로 한 번 따라 해 보세요.
처음부터 내 프로젝트를 바로 만들려고 하면 시행착오가 커집니다.

## 전체 흐름

`初始模板` 폴더에는 `스크립트`와 `프론트엔드 화면` 프로젝트의 초기 파일이 들어 있습니다.
여기서는 `프론트엔드 화면` 기준으로 설명하며, `스크립트`도 흐름은 거의 같습니다.

1. `初始模板/前端界面/新建为src文件夹中的文件夹`를 `src`로 복사
2. 폴더 이름을 원하는 프로젝트명으로 변경
3. AI에게 요구사항을 주고 코드 작성
4. `pnpm build` 실행해 결과를 `dist`로 생성 (`build` = 배포용으로 묶는 작업)
5. 결과 파일을 술집 정규식/스크립트 영역에 반영

### AI에게 요청 예시

```text
기존 규칙을 읽고, 독립적인 프론트엔드 화면 `手机角色状态栏`를 새로 만들어줘.
모바일 스타일의 캐릭터 상태바여야 하며, 테두리/노치/채팅 화면 등을 포함하고
캐릭터 현재 상태를 표시해야 해.
```

## 중요한 주의사항

- 예시처럼 한 번에 큰 요구를 던지기보다, 작은 단위로 나눠서 시키고 바로 테스트하세요.
- AI는 규칙을 가끔 안 읽습니다. 필요하면 `.cursor/rules` 문서를 직접 첨부해 강제하세요.
- 빌드 실패 시 에러 로그를 AI에게 그대로 전달하면 수정 속도가 빨라집니다.

## 빌드 결과 반영 방법

### 프론트엔드 화면

- `dist`의 `.html` 파일 내용을 코드 블록(` ``` `)으로 감싼 뒤,
- 캐릭터 카드 정규식의 `{menuselection}``替换为` 부분에 넣습니다.

(`.map` 파일은 보통 디버깅용이라 초반에는 신경 쓰지 않아도 됩니다.)

### 스크립트

- `dist`의 `.js` 파일 내용을
- 술집 도우미 스크립트의 `{menuselection}``脚本内容`에 넣습니다.

## 왜 실시간 작업이 필요한가

매번 아래를 반복하면 너무 느립니다.

- AI 수정
- `pnpm build` 재실행
- 결과 복사/붙여넣기
- 술집에서 확인

그래서 목표는 다음 2가지입니다.

1. 수정 즉시 자동 빌드
2. 술집에서 최신 결과 자동 로드

## 실시간 반영 설정

### 1) 자동 빌드 (`pnpm watch`)

- `.vscode/launch.json`의 `url`을 내 술집 주소로 수정 (예: `http://localhost:6622`)
- `pnpm build` 대신 `pnpm watch` 실행
(`watch` = 파일 변경을 감시하고 자동 재빌드)

### 2) 술집에서 최신 빌드 자동 로드

- 술집 도우미 메뉴에서 `{menuselection}``酒馆助手 --> 开发 --> 实时监听`의 `{menuselection}``允许监听` 활성화
- `pnpm watch`가 정상 연결되면 WiFi 아이콘이 초록색으로 표시

### 3) 파일을 링크로 공개 (`Go Live`)

- Cursor 우하단 `{menuselection}``Go Live` 클릭
- `port: xxxx` 형태로 바뀌면 로컬 링크 사용 가능 (예: `http://localhost:5500`)
- 예: `dist/前端界面示例/index.html`
  -> `http://localhost:5500/dist/前端界面示例/index.html`

### 4) 정규식에서 링크 로드

기존 정규식의 HTML 부분을 아래처럼 바꿔, 링크에서 직접 로드하게 만듭니다.

```html
<body>
  <script>
    $('body').load('http://localhost:5500/dist/前端界面示例/index.html')
  </script>
</body>
```

## AI가 술집 페이지를 직접 보게 만들기 (MCP)

실시간 반영만으로도 편하지만, MCP를 연결하면 AI가 술집 페이지 상태를 직접 보고 문제를 잡을 수 있습니다.

## 옵션 1 (권장): Chrome DevTools MCP

1. Chrome 최신 버전 설치
2. `chrome://inspect/#remote-debugging` 접속
3. `Allow remote debugging for this browser instance` 체크
4. 체크 후 브라우저 완전 종료 또는 PC 재부팅
5. AI 도구의 MCP 설정에 Chrome DevTools MCP 추가

이후 AI가 가능한 작업:

- 화면 표시/콘솔/네트워크 상태 확인
- 선택한 DOM 요소 정보 읽기 (DOM = 웹페이지 구조 데이터)
- 클릭/입력 등 페이지 조작

프롬프트에 `chrome-devtools-mcp로 술집 페이지 확인`을 명시하면 활용도가 높습니다.

## 옵션 2: Browser MCP

가능하지만 보통 Chrome DevTools MCP보다 사용감이 떨어집니다.

`.cursor/mcp.json` 예시:

```json
{
  "mcpServers": {
    "browsermcp": {
      "command": "pnpx",
      "args": [
        "@browsermcp/mcp"
      ]
    }
  }
}
```

이후 브라우저 확장 설치 후 술집 페이지에서 연결하면 됩니다.

## 디버깅

이 템플릿은 브레이크포인트 디버깅을 지원합니다.

- 코드에 중단점 설정
- `F5`로 디버그 실행
- 해당 줄에서 실행이 멈추면 상태 확인

(브레이크포인트 = 특정 줄에서 실행을 일시정지해 값을 확인하는 기능)

## 최종 작성 절차 (요약)

### 1. 프로젝트 생성

- `src`에 템플릿 기반 프로젝트 생성
- 술집 정규식(또는 스크립트 영역)에 실시간 수정 템플릿 가져오기
- 템플릿 링크를 내 `dist` 링크로 교체

### 2. 개발 진행

- `Go Live` 활성화
- 터미널에서 `pnpm watch` 실행
- 필요 시 MCP 연결
- 프롬프트에 아래를 명시
  - `기존 규칙 읽기`
  - `MCP로 술집 페이지 확인`
  - `결과 검증 절차` (예: 폼 입력 -> 제출 버튼 확인)

### 3. 배포용 결과 게시

- 완료 후 `pnpm watch` 중지 (`Ctrl+C`)
- `pnpm build` 실행
- 결과를 코드 블록으로 감싸 술집 정규식/스크립트에 반영

> 주의: 최종 배포는 `pnpm watch` 결과가 아니라 `pnpm build` 결과를 사용하세요.
> (`watch` 산출물은 실시간 개발용이라 용량이 훨씬 큼)
